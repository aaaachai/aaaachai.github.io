C++文件基础操作

文件介绍

此处只讨论计算机文件，计算机文件是以计算机硬盘为载体存储在计算机上的信息集合。是文件的一种，下列出现的文件一词，均指计算机文件。

我们知道，计算机底层是由二极管和物理电路构成，因此市场上计算机大部分采用二进制存储模式来存储数据，这里的数据就包括文档，图片，音频，视频等。这些数据都以01串，也就是二进制的模式来存储。文件的名字通常是以文件名加上后缀（也叫文件扩展名），文件名用于区别同一类型下不同文件，后缀用于指示文件的类型。本文只考虑对文本文档文件的操作，即以.txt为后缀的文件，其他类型文件有各自独有的处理方法，不过基础原理都一致。有兴趣的可以自行查阅资料。

C++中的文件流

C++ 标准库中专门提供了 3 个类用于实现文件操作，它们统称为文件流类，这 3 个类分别为：

  • ifstream：专用于从文件中读取数据；

  • ofstream：专用于向文件中写入数据；

  • fstream：既可用于从文件中读取数据，又可用于向文件中写入数据。

这三个文件流都位于fstream头文件中，使用之前需要将这个头文件包含进来。同时我们需要自己创建文件流对象来对不同文件进行操作。

打开文件

在对文件进行操作时，当创建好文件流对象时，要想继续对文件进行操作，那么就必须打开文件，C++中使用open（）函数，他的函数原型是void open( const char *filename );，如果使用ofstream文件流创建了一个ofs对象，那么我们可以使用ofs.open("demo.txt");来打开一个文件，当然这个对象只能将数据写入文件中。C++中还提供了各种模式标志来对打开文件做了更多的定义。

模式标志 描述
ios::in 读方式打开文件
ios::out 写方式打开文件
ios::trunc 如果此文件已经存在, 就会打开文件之前把文件长度截断为0
ios::app 尾部最加方式(在尾部写入)
ios::ate 文件打开后, 定位到文件尾
ios::binary 二进制方式(默认是文本方式)

当你想使用多种打开方式时，可以使用位操作|来对模式标志进行组合，例如ofs.open("demo,txt",ios::out|ios::app);当打开文件以后，我们还需要对是否成功打开文件进行判断，使用is_open()函数，通常我们采用if语句来判断

文件指针

值得一说的是，C++也支持C的文件操作，可以利用FILE结构体来实现对文件的操作。

感兴趣的可以查阅相关资料。值得注意的是，文件的读写操作是基于文件指针进行的，什么意思呢，你可以把文件看作一个很大的数组，里面的一个字符就是一个数组元素，而文件指针指向该元素。当我们读取文件的时候，文件指针是跟着往后走的，所以如果我们想要二次读取文件，你就需要将文件指针挪到首地址来，又或者你想跳过某些数据，此时你也可以通过移动文件指针来实现。在C++中有两个成员函数能实现文件指针的移动。seekg()和seegp()。在读取文件时，使用seekg(),在写文件时使用seekp()。

seekg()函数

seekg(offset, place);

这个输入流类的成员函数的名字 seekg 由两部分组成。首先是 seek（寻找）到文件中的某个地方，其中是 “g” 表示 “get”，指示函数在输入流上工作，因为要从输入流获取数据。

要查找的文件中的新位置由两个形参给出：新位置将从由 place 给出的起始位置开始，偏移 offset 个字节。offset 形参是一个 long 类型的整数，而 place 可以是 ios 类中定义的 3 个值之一。起始位置可能是文件的开头、文件的当前位置或文件的末尾，这些地方分别由常量 ios::beg、ios::cur 和 ios::end 表示。比如

ifs.seekg(3L,ios::beg)

就是表示从文件头开始，偏移三个字节（L表示这个数据是long类型）

seekp()函数

原理同seekg一致，只不过这个函数用于输出流，即对文件进行写操作时的文件指针移动，可以实现指定位置的插入，追加等功能。

本文只是浅略介绍了C++中的文件操作。同时C++还支持对二进制文件的读写操作。这样使得数据更保密。有兴趣也可以进行了解。
