力扣-剑指 Offer 30.包含min函数的栈(最小栈)

发表于2023-05-23|更新于2023-05-23
|阅读量:

题目描述

设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈。

实现 MinStack 类:

MinStack() 初始化堆栈对象。
void push(int val) 将元素val推入堆栈。
void pop() 删除堆栈顶部的元素。
int top() 获取堆栈顶部的元素。
int getMin() 获取堆栈中的最小元素。

来源：力扣（LeetCode）
链接：https://leetcode.cn/problems/min-stack

辅助栈

辅助栈是最容易想到的办法，即使用两个栈来保存数据信息，一个栈用于保存数据的原始信息，另一个栈则用来动态的记录栈中元素的最小值。具体算法流程如下

  • push() 第一个栈正常push，将要push的元素x与第二个栈的栈顶元素比较，如果比栈顶元素(未插入前整个栈中的最小值)小，则将元素x，push进第二个栈中。
  • pop() 第一个栈正常pop(),将要弹出的元素x与第二个栈的栈顶比较，如果两者相等，则将第二个栈的栈顶元素一并弹出以维护最小值。
  • getMin()， 返回第二个栈的栈顶即可。

复杂度

时间复杂度 ：O(1) ，压栈，出栈，获取最小值的时间复杂度都为 O(1) 。
空间复杂度 ：O(N) ，包含 N 个元素辅助栈占用线性大小的额外空间。

衍生

根据这种思想，也可以在stack中就存储pair<int,int>对象，只用一个栈来完成要求，不过还是使用了额外的空间。也可以使用链表的数据结构，链表节点包含两个成员，分别为原始数据和当前栈的最小值，实现思路也是一致的。

差值法

算法原理

在上一个算法中，为什么需要一个辅助栈来保存最小值？因为如果我们只使用一个变量来保存最小值，虽然在数据入栈的过程中并不会出错，但是当数据进行弹栈时，如果弹出的数据就是这个栈的最小值，那么为了维护栈最小值，我们就必须选择第二小的元素来充当最小值，如果没有辅助栈的存在，我们就必须要对剩下的栈进行一次查找最小值的操作，而这个操作是o(n)的复杂度。

所以解决问题的关键就是如何能保存第二小的数的信息,在差值法中，我们不对数据本身进行存储，而是将其与minNum(当前栈的最小值)的差值(当前元素减去栈的最小值)进行存储。很容易看到，如果要压栈的元素比当前栈的最小值大，那么存储到栈中的元素就是大于0的，而如果要压栈的元素比当前栈的最小值小，那么此时这个元素在压栈以后，就会成为这个栈的新的最小值。这里需要注意的是，这个元素压入栈时，存的数据是这个元素减去以前的minNum值，因此存储的会是一个负数，而这也正好可以作为在弹栈时更新minNum值的条件。

通过差值存储数据，我们实现了类似于使用辅助栈动态存储最小值的功能，在每次栈存储的数据为负数的时候，就说明这个数据的插入，会打破栈中最小值的地位。即这个数据是插入后新栈的最小值。根据这个条件，就可以在压栈，弹栈的时候，对minNum值进行更新维护。

算法过程

  • push() 压入第一个数据时，直接push(0),然后令minNum等于第一个元素的值。如果栈不为空，则压入（x-minNum），如果压入的数据比minNum小，则更新minNum的值，这里需要注意的是，无论压入数据是否比minNum小还是大，都先push(x-minNum),然后再进行minNum的更新。
  • pop() 如果栈顶元素大于0，直接pop()。如果栈顶元素小于0，则需要更新minNum值，令 minNum = minNum - st.top()。解释一下，在执行这条代码前，minNum指的是在即将要弹出的数据(因为push操作保证了只有当要插入的数据能成为新的最小值时，栈中的数据才会为负数)又因为在插入时，此处栈顶的元素使用的是(x-minNum(老的)），所以为了得到老的minNum，我们需要将使用x减去st.top().而x是当前的minNum值，所以整个代码就是 minNum = minNum - st.top()。
  • top() 如果栈顶元素大于0，则返回st.top()+minNum，如果栈顶元素小于0，则直接返回minNum，原因如上。
  • getMin() 直接返回minNum
